**Goal:** Inline rename is safely cancellable and never double-mounted, and folder Duplicate actually persists a copy instead of firing a success toast on nothing.

## Phase 73: drawing-mode-transforms

**Goal:** Single-object selections in drawing mode expose Fabric.js scale and rotation handles, matching multi-select behavior.

**Requirements:** DRAW-01
**Plans:** 1 plan (1 wave)

- [x] **73-01** — Restore `evented = true` on objects when tool is `'select'` (DRAW-01). Root cause: the tool-switch effect force-sets `evented = false` on every object before each tool change (correct — non-select tools need a clean canvas); the `'select'` case restored `selectable = true` but **not** `evented = true`. With `evented = false`, Fabric's single-object control-handle hit-testing short-circuits to the canvas-level move handler. Multi-select worked because Fabric's `ActiveSelection` rubberband wraps objects in a separate evented group. One-line fix in `src/components/viewer/AnnotationCanvas.tsx:163`. Commit `e975334e`.

**Files:** `src/components/viewer/AnnotationCanvas.tsx`

**Verification:** `human_needed` — 4/4 code-level must-haves pass; 5 runtime behaviors require browser verification (handle scale, handle rotate, multi-select regression, drawing tools regression, read-only display regression).

## Key design decisions

| Decision | Outcome | Phase |
|----------|---------|-------|
| Singleton `ContextMenuProvider` holds one `{key, position, items} \| null` state instead of coordinating N local states | ✓ Makes two-menus-open physically impossible, not merely forbidden | 70-01 |
| `buildFileBrowserActions()` pure-data factory; icons supplied by consumer | ✓ Prevents layered coupling; lucide-react stays out of the data layer | 70-02 |
| Three layered defenses on FolderCard click-through (target guard + suppressRef + mouseDown preventDefault) | ✓ Defense in depth — each catches a different synthetic-event path | 70-02 |
| Parent `onMouseMove` with `pointer-events-none` on overlay layers, instead of exclusion zone math | ✓ Simpler than computing a 48×48 dead zone; scrub still works via bubble | 71-01 |
| `RenameController` context instead of document-level custom events | ✓ Same pattern as Phase 70's context-menu singleton — familiar | 72-01 |
| Fix `deepCopyFolder` at the helper, not at the route | ✓ Side-effect: repairs "Copy to folder" too | 72-02 |
| One-line `evented = true` fix instead of a broader selection-lifecycle refactor | ✓ CLAUDE.md §3 (surgical changes) honored; zero collateral risk | 73-01 |
