## Goal

Every asset delivered through a review link is automatically stamped with Meta-required XMP attribution (`http://ns.attribution.com/ads/1.0/`) using server-side `exiftool-vendored`, matching the `scf-metadata` desktop app 1:1. Stamps are cached per asset, invalidated on rename / new version, and served exclusively to review-link guests. Internal viewers always receive the original.

## Key Architecture Decisions

- **Fully async stamp jobs** — POST never blocks on stamping; Vercel 60s budget cannot absorb inline stamp of even one large-file video
- **Asset-level GCS path** (`projects/{pid}/assets/{aid}/stamped{ext}`) — deterministic, shared across review links, overwrite-safe across re-stamps
- **Attrib append-semantics** — read existing entries, re-stamp `Data` on each, append new entry; matches reference 1:1
- **Per-request ExifTool instance** with `maxProcs:1, maxTasksPerProcess:1` + `await et.end()` in finally — no zombie perl across warm container reuses
- **`decorate()` fallback to original URL** — guests always see working content; stamp absence/failure/staleness never produces a 503
- **Concurrency dedup** via Firestore jobs-collection query at route entry — idempotent, Firestore-coordinated
- **Hardcoded `FB_ID=2955517117817270`, `DATA='|{"Company":"Ready Set"|}'`, `META_TZ='America/New_York'`** — matches reference app; future milestone may extract to `project.metaConfig`
