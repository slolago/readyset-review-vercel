# Requirements Archive: v2.2 Dashboard & Annotation UX Fixes

**Shipped:** 2026-04-21
**Source:** 9 UI/UX bugs reported from hands-on use of the dashboard file browser, inline rename flow, and drawing-mode canvas.

## Requirements (all complete)

### Context menu behavior

- [x] **CTX-02** → Phase 70: In grid view, a right-click context menu on an asset or folder stays inside the viewport regardless of position.
- [x] **CTX-03** → Phase 70: Left-click outside / Escape / right-click-other closes; only one menu open at a time.
- [x] **CTX-04** → Phase 70: Right-click menu exposes full action set; floating bottom bar is a curated shortcut; consistent across asset/folder/mixed.
- [x] **CTX-05** → Phase 70: Right-clicking a folder opens the menu (never the folder's default open behavior); same menu as three-dots.

### Grid / list affordances

- [x] **VIEW-01** → Phase 71: List/grid view toggle functional when current folder contains only folders.
- [x] **VIEW-02** → Phase 71: Asset three-dots button reachable in grid view; hover preview doesn't consume pointer events over it.

### Inline edit + mutations

- [x] **EDIT-01** → Phase 72: Click-away cancels rename; only one rename input active at a time.
- [x] **FS-01** → Phase 72: Folder Duplicate creates a real duplicate that appears in listings (not just a success toast).

### Drawing mode

- [x] **DRAW-01** → Phase 73: Single-object selection in drawing mode exposes working Fabric.js scale + rotation handles.
