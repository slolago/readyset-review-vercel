## Known Deviations (Intentional)

1. **Per-request ExifTool instance** instead of reference app's `-stay_open True` singleton — serverless requires this (PITFALLS.md HIGH finding)
2. **Config file passed to `et.write()` args**, not the ExifTool constructor — reference app passed via constructor, which conflicts with `-stay_open` protocol in exiftool-vendored; per-write is the documented pattern
3. **Streaming GCS upload** via `uploadStream()` — reference app writes to local disk only; server-side pipeline requires GCS round-trip; naive `uploadBuffer` OOMs on 500MB+ source
4. **Asset-level stamp caching** (not re-stamp per review link) — reference app is per-delivery; server-side caches because fields are deterministic from filename + constants
5. **Active null-clearing on rename** — reference app doesn't have this concept; added for spec literalism + cached-URL hygiene
