5. The component body is a direct extraction of FolderBrowser.tsx lines 492-535. The render logic:
   - Outer element: `<nav className="flex items-center gap-1 text-sm overflow-x-auto">`
   - Map over `items` array. For each item at index `i`:
     - Compute `isLast = i === items.length - 1`
     - Compute `isRoot = i === 0`
     - Compute `href`: if `item.id` is truthy, use `/projects/${projectId}/folders/${item.id}`, otherwise `/projects/${projectId}`
     - Wrap in `<span key={item.id ?? 'root'} className="flex items-center gap-1 flex-shrink-0">`
     - If `i > 0`, render separator: `<ChevronRight className="w-4 h-4 text-frame-textMuted" />`
     - If `isLast` (active crumb): render `<span className="flex items-center gap-1.5 text-white font-medium">` with the name. If also `isRoot`, prepend the Home icon badge.
     - If NOT `isLast` (clickable crumb): render `<Link href={href} className="flex items-center gap-1.5 text-frame-textSecondary hover:text-white transition-colors">` with the name. If also `isRoot`, prepend the Home icon badge.
   - The Home icon badge markup (used in both isLast and non-isLast branches):
     ```tsx
     <div
       className="w-5 h-5 rounded flex items-center justify-center"
       style={{ backgroundColor: projectColor + '20', color: projectColor }}
     >
       <Home className="w-3 h-3" />
     </div>
     ```
     Note: use `projectColor` prop instead of the `color` variable that FolderBrowser used.

6. No other logic — no state, no effects, no hooks. Pure presentational component.
