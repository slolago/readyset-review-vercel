Create `src/app/api/assets/unstack-version/route.ts` with a single `POST` export.

**Request body:** `{ assetId: string }`

**Logic (follow merge-version/route.ts auth pattern exactly):**

1. `getAuthenticatedUser(request)` — return 401 if null.
2. Parse `assetId` from body — return 400 if missing.
3. `const db = getAdminDb();`
4. Fetch the target asset doc: `db.collection('assets').doc(assetId).get()` — return 404 if not exists.
5. `const asset = assetDoc.data() as any;`
6. Auth check: `canAccessProject(user.id, asset.projectId)` — return 403 if false.
7. Resolve group: `const groupId: string = asset.versionGroupId || assetId;`
8. **Self-check:** If `groupId === assetId` and there are no other members (asset is already standalone), return `{ error: 'Asset is not part of a version stack' }` with status 400.
9. Query all group members: `db.collection('assets').where('versionGroupId', '==', groupId).get()`
10. Build members array: `{ id: string; version: number }[]` from query docs.
11. Root inclusion guard (same as merge-version): if `!members.some(m => m.id === groupId)`, fetch root doc explicitly and push it.
12. Remove the target asset from the members array: `const remaining = members.filter(m => m.id !== assetId);`
13. If `remaining.length === 0`, this was the only other member — should not happen due to step 8, but guard anyway.
14. **Batch write:**
    ```
    const batch = db.batch();
    // Detach target — becomes standalone
    batch.update(db.collection('assets').doc(assetId), {
      versionGroupId: assetId,   // CRITICAL: set to own id, never null
      version: 1,
    });
    // Re-compact remaining 1..N
    remaining.sort((a, b) => a.version - b.version);
    remaining.forEach((m, i) => {
      batch.update(db.collection('assets').doc(m.id), { version: i + 1 });
    });
    await batch.commit();
    ```
15. Return `NextResponse.json({ unstacked: assetId, remaining: remaining.length })` with status 200.
16. Wrap in try/catch — log error, return 500.

**Imports:** `NextRequest`, `NextResponse` from `next/server`; `getAuthenticatedUser`, `canAccessProject` from `@/lib/auth-helpers`; `getAdminDb` from `@/lib/firebase-admin`.

Create `src/app/api/assets/reorder-versions/route.ts` with a single `POST` export.

**Request body:** `{ orderedIds: string[] }`

**Logic:**

1. `getAuthenticatedUser(request)` — return 401 if null.
2. Parse `orderedIds` from body — return 400 if missing, not an array, empty, or contains non-strings.
3. `const db = getAdminDb();`
4. **Firestore transaction** (STATE.md mandate — NOT a batch):
    ```
    await db.runTransaction(async (tx) => {
      const refs = orderedIds.map((id: string) => db.collection('assets').doc(id));
      const docs = await Promise.all(refs.map((r) => tx.get(r)));

      // Verify all docs exist
      for (const doc of docs) {
        if (!doc.exists) throw new Error(`Asset ${doc.id} not found`);
      }

      // Auth check on first doc
      const firstAsset = docs[0].data() as any;
      // (auth check done outside transaction — see below)

      // Verify all belong to same versionGroupId
      const groupIds = new Set(
        docs.map((d) => {
          const data = d.data() as any;
          return data.versionGroupId || d.id;
        })
      );
      if (groupIds.size > 1) throw new Error('Cross-group reorder not allowed');

      // Write new version numbers 1..N
      docs.forEach((doc, i) => {
        tx.update(doc.ref, { version: i + 1 });
      });
    });
    ```
5. **Auth check:** Before the transaction, fetch the first asset to get `projectId`, then call `canAccessProject(user.id, projectId)` — return 403 if false. This avoids doing the auth check inside the transaction (which would add a read that complicates retries).
6. Return `NextResponse.json({ reordered: orderedIds.length })` with status 200.
7. Catch transaction errors: if message includes "Cross-group" return 400, if "not found" return 404, else 500.

**Full flow order:**
1. Auth user
2. Validate orderedIds
3. Fetch first asset for projectId, check canAccessProject
4. Run transaction (read all, verify same group, write versions)
5. Return success

**Imports:** Same as unstack-version route.
