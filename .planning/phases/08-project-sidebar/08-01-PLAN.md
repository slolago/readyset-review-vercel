  <action>
Create `src/hooks/useProjectTree.ts` as a 'use client' hook.

Export a `ProjectTreeNode` interface:
```typescript
export interface ProjectTreeNode {
  project: Project;
  folders: Folder[];          // top-level folders (parentId === null)
  foldersLoaded: boolean;     // true once fetch has completed for this project
  expanded: boolean;          // UI collapse/expand state
}
```

Export `useProjectTree()` hook that:
1. Calls `useProjects()` (imported from ./useProject) to get the project list
2. Maintains `treeNodes: ProjectTreeNode[]` state — one entry per project
3. Whenever `projects` from useProjects() changes, syncs treeNodes (add new projects, preserve existing expanded/foldersLoaded state by project id)
4. Exposes `toggleProject(projectId: string)`: flips `expanded`, and if `foldersLoaded` is false for that project, fetches top-level folders for it via `GET /api/folders?projectId={id}&parentId=null` with auth token (use `getIdToken()` from useAuth), then sets `folders` and `foldersLoaded: true`
5. Returns `{ treeNodes, loading, toggleProject }`

Do NOT eagerly load all folders on mount — only load when a project is expanded for the first time. This avoids N+1 fetches at startup.
