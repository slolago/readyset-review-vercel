# Plan 26-01: File Info Tab in Asset Viewer Sidebar

## Objective

Add an "Info" tab to the asset viewer sidebar alongside the existing Comments tab. The Info tab shows technical metadata: filename, MIME type, file size, duration, resolution, aspect ratio, uploaded by, upload date, and version number. A new `FileInfoPanel` component renders the metadata rows. When a field is not available (e.g. FPS, width/height for audio) it shows "â€”".

## Context

- `src/app/(app)/projects/[projectId]/assets/[assetId]/page.tsx` â€” AssetViewerPage; passes `asset` and `comments` to `CommentSidebar`. Tab state lives here or in CommentSidebar.
- `src/components/viewer/CommentSidebar.tsx` â€” w-80 sidebar with Comments header, list, and input. Needs a tab bar added at the top that toggles between Comments and Info panels.
- `src/types/index.ts` â€” `Asset` interface has: `name`, `mimeType`, `size`, `duration?`, `width?`, `height?`, `uploadedBy`, `createdAt`, `version`.
- No new API calls needed â€” all metadata is on the `asset` object already loaded by `useAsset`.

### Task 2: Add tab bar to CommentSidebar and wire FileInfoPanel

**Type:** auto

**Done when:** CommentSidebar renders a two-tab bar (Comments / Info) at the top. Clicking "Info" hides the comments list/input and shows `FileInfoPanel`. Clicking "Comments" restores the current view. Tab state is internal to CommentSidebar.

**Implementation:**

1. In `CommentSidebar.tsx` add:
   - `const [activeTab, setActiveTab] = useState<'comments' | 'info'>('comments');`
2. Replace the current header `<div>` with a tab bar containing two tabs:
   - "Comments" tab: `MessageSquare` icon + "Comments" text + comment count badge (existing logic).
   - "Info" tab: `Info` icon (from `lucide-react`) + "Info" text.
   - Active tab style: `border-b-2 border-frame-accent text-white`.
   - Inactive tab style: `text-frame-textMuted hover:text-white`.
   - Tab bar container: `flex border-b border-frame-border` (replaces the old header).
3. Conditionally render:
   - When `activeTab === 'comments'`: current comments list + resolve toggle button + input section (unchanged).
   - When `activeTab === 'info'`: `<FileInfoPanel asset={asset} />` fills the remaining space with `overflow-y-auto`.
4. The Filter/resolve toggle button is only shown when `activeTab === 'comments'`.
5. Import `FileInfoPanel` from `./FileInfoPanel`.
6. Import `Info` from `lucide-react`.
