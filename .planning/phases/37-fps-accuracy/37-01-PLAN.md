# Plan 37-01: Snap raw rVFC measurement to nearest standard frame rate

## Goal
FPS in the info tab reflects the file's true frame rate. `requestVideoFrameCallback` can produce jitter (e.g., 30fps file reads as 31fps). Snap the raw measurement to the nearest standard rate if within ±0.6fps.

## Context
- **File**: `src/hooks/useAssets.ts`
- **Current line 91**: `const fps = elapsed > 0 ? Math.round(frameCount / elapsed) : undefined;`
- **Problem**: `Math.round(frameCount / elapsed)` is a raw integer — jitter causes 30fps → 31fps, 29.97fps → 30fps
- **Standard rates**: [23.976, 24, 25, 29.97, 30, 48, 50, 59.94, 60]
- **Tolerance**: ±0.6fps — if raw value is within 0.6 of a standard rate, snap to it

## Implementation

### Step 1 — Add snapToStandardFps helper before extractVideoMetadata

```ts
const STANDARD_FRAME_RATES = [23.976, 24, 25, 29.97, 30, 48, 50, 59.94, 60];
const FPS_SNAP_TOLERANCE = 0.6;

function snapToStandardFps(raw: number): number {
  let closest = raw;
  let minDelta = Infinity;
  for (const std of STANDARD_FRAME_RATES) {
    const delta = Math.abs(raw - std);
    if (delta < minDelta) {
      minDelta = delta;
      closest = std;
    }
  }
  return minDelta <= FPS_SNAP_TOLERANCE ? closest : raw;
}
```

### Step 2 — Replace Math.round line (line 91) with snap

Change:
```ts
const fps = elapsed > 0 ? Math.round(frameCount / elapsed) : undefined;
```
To:
```ts
const rawFps = elapsed > 0 ? frameCount / elapsed : undefined;
const fps = rawFps !== undefined ? snapToStandardFps(rawFps) : undefined;
```
