# Plan 28-01: POST /api/assets/merge-version

## Objective

Create the `POST /api/assets/merge-version` API route that merges a source asset's entire version group into a target asset's version group. The merge is atomic (single Firestore batch write) and renumbers version numbers to avoid collisions. Self-merge and same-group merge return 400. Route requires authentication.

## Context

- `src/app/api/assets/route.ts` — Existing assets endpoint; pattern for auth + Firestore usage.
- `src/app/api/assets/copy/route.ts` — Batch write pattern for multi-version operations.
- `src/app/api/assets/[assetId]/route.ts` — Version group fetch pattern: query by `versionGroupId`, fall back to root asset ID.
- `src/lib/auth-helpers.ts` — `getAuthenticatedUser`, `canAccessProject`.
- `src/lib/firebase-admin.ts` — `getAdminDb()`.

## Tasks

### Task 1: Create merge-version API route

**Type:** auto

**Done when:** `src/app/api/assets/merge-version/route.ts` exists; accepts `POST { sourceId, targetId }`; fetches source group and target group; validates no self-merge / same-group merge; renumbers source members starting at `maxTargetVersion + 1`; updates `versionGroupId` + `version` on each source member; performs all writes in a single Firestore batch; returns `{ merged: number }`.

**Implementation:**

1. Create directory `src/app/api/assets/merge-version/` and file `route.ts`.
2. Import: `NextRequest`, `NextResponse` from `'next/server'`; `getAuthenticatedUser`, `canAccessProject` from `'@/lib/auth-helpers'`; `getAdminDb` from `'@/lib/firebase-admin'`.
3. Export `POST` handler:
   a. Authenticate: `const user = await getAuthenticatedUser(request)` — return 401 if falsy.
   b. Parse body: `const { sourceId, targetId } = await request.json()`.
   c. Validate: both present — return 400 if missing.
   d. Self-merge check: `if (sourceId === targetId)` return 400 `{ error: 'Cannot merge asset with itself' }`.
   e. Fetch source doc from `assets` collection — return 404 if not found.
   f. Fetch target doc from `assets` collection — return 404 if not found.
   g. Auth check: `canAccessProject(user.id, source.projectId)` — return 403 if no access. (Both assets assumed in same project; source check is sufficient.)
   h. Resolve source group ID: `sourceGroupId = source.versionGroupId || sourceId`.
   i. Resolve target group ID: `targetGroupId = target.versionGroupId || targetId`.
   j. Same-group check: `if (sourceGroupId === targetGroupId)` return 400 `{ error: 'Assets are already in the same version stack' }`.
   k. Fetch all docs in source group: query `assets` where `versionGroupId == sourceGroupId`. If query returns 0 docs, also fetch the source doc itself (root asset may not have `versionGroupId` set).
   l. Fetch all docs in target group: query `assets` where `versionGroupId == targetGroupId`. If query returns 0 docs, also fetch the target doc itself.
   m. Calculate `maxTargetVersion`: `Math.max(...targetMembers.map(d => d.version || 1))`.
   n. Sort source members by version ascending.
   o. Build batch: for each source member (index `i`):
      - `batch.update(db.collection('assets').doc(member.id), { versionGroupId: targetGroupId, version: maxTargetVersion + 1 + i })`.
   p. `await batch.commit()`.
   q. Return `NextResponse.json({ merged: sourceMembers.length }, { status: 200 })`.
4. Wrap main logic in try/catch; return 500 on unexpected errors.
