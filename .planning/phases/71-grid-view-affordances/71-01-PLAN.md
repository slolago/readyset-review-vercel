<objective>
Make the list/grid toggle work in folders-only folders (VIEW-01) and make the asset three-dots button reachable past the hover preview (VIEW-02). Both are small, surgical fixes to existing markup; no new components, no API changes.

Purpose:
- VIEW-01 — The view toggle is already unconditionally rendered, but the folders block in `FolderBrowser.tsx` is hardcoded to a responsive CSS grid regardless of `viewMode`. When the current folder has only folders, switching to list does nothing visible because the list-mode branch only renders `AssetListView` and that component bails out on empty assets. The fix is to branch folder rendering on `viewMode`: grid → existing `<div className="grid grid-cols-…">` of `FolderCard`s; list → a table matching the asset list table styling, with a single "Name / Date" row per folder and the same three-dots action button as the existing FolderCard.
- VIEW-02 — The sprite-strip hover overlay (`z-[1]`) and scrub-progress bar (`z-[2]`) sit above the three-dots actions wrapper, which has no explicit z-index. Result: the overlay eats pointer events over the top-right corner where the button lives, and on some paths the button is visually occluded too. Raise the actions wrapper to `z-20` (matching FolderCard's three-dots) and mark the sprite overlay + scrub bar as `pointer-events-none` (they are purely visual — the `onMouseMove` handler lives on the parent thumbnail div and still receives events via bubbling).

Output: Folders render as rows in list view (VIEW-01); the three-dots button is always reachable on asset cards (VIEW-02). Both fixes parity-match the existing folder card three-dots behavior.
</objective>
