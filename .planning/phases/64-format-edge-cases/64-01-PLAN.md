# Phase 64 Plan 01: Format Edge Cases

## Objective

Make the export, image-metadata, and sprite pipelines robust against real-world format variety. Stop rejecting `.mov+h264` on the export copy path. Reap ghost `running` jobs left behind by SIGKILL'd functions. Fall back to `ffprobe` for HEIC/AVIF dimensions. Stop clustering sprite frames on very short/long videos.

## Tasks

1. FMT-01 — widen the export copy-path container matrix in `/api/exports/route.ts` to accept `mov+h264+aac` (iPhone / Premiere exports) in addition to `mp4`. Non-h264 sources keep the re-encode path.
2. FMT-02 — add `sweepStaleJobs(olderThanMs = 120000)` in `src/lib/jobs.ts` that batch-flips `status=running, startedAt < cutoff` jobs to `failed` with a SIGKILL error. Call it lazily from the jobs GET endpoint.
3. FMT-03 — add an `ffprobe` fallback in `src/lib/image-metadata.ts` after the two existing paths (range fetch, bounded full download). Promote `resolveFfprobe()` into `src/lib/ffmpeg-resolve.ts` so it can be reused without a second implementation.
4. FMT-04 — adaptive sprite frame spacing in `/api/assets/[id]/generate-sprite/route.ts`: clamp span for `<3s` videos and cap sampling window at 2h for very long videos. Keep the tile exactly 20 slots wide (pad repeats of the last timestamp) so the client hover-scrub math stays intact.

## Verification

- `tsc --noEmit` clean
- `vitest run` clean
- Each task in its own commit; no push.

## Anti-Scope

- No HDR color pipeline work.
- No hardware-decode paths.
- No on-the-fly transcode UI.
- No server-side cron for stale-job sweeping — on-read lazy sweep is sufficient for now.
