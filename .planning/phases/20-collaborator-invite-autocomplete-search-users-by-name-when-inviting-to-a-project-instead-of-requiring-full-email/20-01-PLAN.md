**1. Create `src/app/api/users/search/route.ts`**

Create a GET endpoint that accepts query params `q` (search string) and `exclude` (comma-separated user IDs to omit).

- Import `getAuthenticatedUser` from `@/lib/auth-helpers` and `getAdminDb` from `@/lib/firebase-admin`.
- Return 401 if not authenticated.
- Trim `q`. If `q.length < 2`, return `{ users: [] }` (require at least 2 characters — per research recommendation P20-02).
- Convert `q` to lowercase for the query variable `qLower`.
- Run TWO Firestore prefix queries in parallel using `Promise.all`:
  - Name query: `db.collection('users').orderBy('name').where('name', '>=', q).where('name', '<', q + '\uf8ff').limit(8)`
  - Email query: `db.collection('users').orderBy('email').where('email', '>=', qLower).where('email', '<', qLower + '\uf8ff').limit(8)`
- Note on case sensitivity: The name query uses the raw `q` as-is. Firestore range queries are case-sensitive. This is acceptable for v1 — names are stored as entered during Google sign-in (typically Title Case). If the user types "al" it will match "alice" but not "Alice". Document this as a known limitation. A `nameLower` field can be added later.
- Merge results, deduplicating by `doc.id`. Skip any `doc.id` present in the `exclude` set.
- Return `{ users: [{ id, name, email }] }` capped at 8 results.
- Wrap in try/catch, return 500 on error.

**2. Create `src/components/ui/UserSearchCombobox.tsx`**

Build a controlled combobox component with these props:

```typescript
interface UserResult { id: string; name: string; email: string; }
interface UserSearchComboboxProps {
  onSelect: (user: UserResult) => void;
  onClear: () => void;       // called when input changes after a selection (invalidates selected user)
  exclude?: string[];         // user IDs to hide from results
  placeholder?: string;
  disabled?: boolean;
}
```

Implementation details:
- Use `useState` for `query` (string), `results` (UserResult[]), `open` (boolean), `loading` (boolean).
- Use `useRef` for `timerRef` (debounce timer) and `versionRef` (number, for stale response detection — per P20-05).
- Import `useAuth` from `@/hooks/useAuth` for `getIdToken`.
- Import `Input` from `@/components/ui/Input`, `Avatar` from `@/components/ui/Avatar`.
- Import `Search`, `Loader2` from `lucide-react`.
- On input change: update `query`, call `onClear()` (to invalidate any prior selection), clear previous timer, set new 250ms debounce timer that calls `search(value)`.
- `search(q)` function: increment `versionRef.current`, capture version. If `q.trim().length < 2`, clear results and return. Set `loading=true`. Fetch `GET /api/users/search?q=${encodeURIComponent(q)}&exclude=${exclude.join(',')}` with Authorization header. If `versionRef.current !== captured version`, discard (stale). Otherwise set results and `open=true`. Set `loading=false` in finally.
- On item click (use `onMouseDown` not `onClick` to fire before input blur): set `query` to `user.name`, close dropdown, clear results, call `onSelect(user)`.
- On input focus: if results exist, open dropdown.
- On input blur: `setTimeout(() => setOpen(false), 150)` to allow mousedown on items to register first.
- Show a hint text "Type at least 2 characters" below the input when `query.length === 1`.
- Render the `Input` with a `Search` icon prop. When `loading` is true, show `Loader2` with `animate-spin` instead of Search.
- Dropdown: absolute-positioned `ul` with `z-50`, `mt-1`, `w-full`, `bg-frame-surface`, `border border-frame-border`, `rounded-xl`, `shadow-lg`, `overflow-hidden`, `max-h-64 overflow-y-auto`.
- Each item: `li` with `px-4 py-2.5`, `cursor-pointer`, `hover:bg-frame-hover`, flex row with `Avatar` (size="sm"), name (text-sm text-white), email (text-xs text-frame-textMuted).
- If search returned 0 results and query.length >= 2 and not loading: show "No users found" in the dropdown area.

Export `UserSearchCombobox` and the `UserResult` type as named exports.
