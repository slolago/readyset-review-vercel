<objective>
Fix context menu viewport clipping (CTX-02) and enforce the single-open + click-away + Escape invariants (CTX-03) by upgrading `ContextMenu.tsx` with a measured post-render flip and introducing a lightweight provider/controller so only one context menu can be open across the whole file browser.

Purpose: Right-clicks near the right/bottom edge currently clip menu items off-screen because the flip math uses a hardcoded 200×rows estimate instead of the real rendered rect; and four separate `useState<{x,y}|null>` context-menu instances (AssetCard, AssetListView row, FolderCard, FolderBrowser canvas) mean two menus can visually co-exist if events race. One controller = one truth.

Output: An updated `ContextMenu` that measures itself after mount and flips/clamps to the viewport, and a `ContextMenuProvider` whose `open(key, {x,y}, items)` / `close()` API guarantees a single open menu.
</objective>
