# Plan 41-01: Add persistent download button to asset viewer player controls

## Goal
An always-visible "Download" button appears in the video player controls — no hover required. Visible to review link guests when `allowDownloads` is true.

## Context
- **VideoPlayer**: `src/components/viewer/VideoPlayer.tsx` — add `downloadUrl?: string` prop; render Download button in controls bar when provided
- **Review page**: `src/app/review/[token]/page.tsx` — pass `downloadUrl` to `<VideoPlayer>` when `data.reviewLink.allowDownloads` is true
- **Internal viewer**: already has a permanent download button in the header bar (no change needed)

## Implementation

### VideoPlayer.tsx
1. Add `Download` icon import + `forceDownload` util import
2. Add `downloadUrl?: string` to `VideoPlayerProps`
3. Destructure `downloadUrl` in component
4. Add Download button in controls bar before Fullscreen button

### review/[token]/page.tsx
1. Pass `downloadUrl` prop to `<VideoPlayer>` when `data.reviewLink.allowDownloads`:
   ```tsx
   downloadUrl={
     data.reviewLink.allowDownloads
       ? ((selectedAsset as any).downloadUrl ?? (selectedAsset as any).signedUrl)
       : undefined
   }
   ```

## Verification
1. `npm run build` — zero errors
2. Open a review link with `allowDownloads: true`, click an asset → Download button visible in player controls
3. Review link with `allowDownloads: false` → no Download button
4. Clicking Download triggers the download
