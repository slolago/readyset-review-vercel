Purpose: GCS signed URLs with no `Content-Disposition` header are opened inline by the browser. The `download` attribute on an `<a>` tag is ignored for cross-origin URLs (GCS is `storage.googleapis.com`). Setting `responseDisposition: attachment` in the signed URL config causes GCS to serve the header server-side, so the browser saves the file regardless of origin.

Output: `generateDownloadSignedUrl` in `gcs.ts`; `downloadUrl` field returned by review-link API; download buttons in review page, AssetCard, and AssetListView updated.

```typescript
export async function generateDownloadSignedUrl(
  gcsPath: string,
  filename: string,
  expiresInMinutes: number = 720
): Promise<string> {
  const storage = getStorage();
  const bucket = storage.bucket(BUCKET_NAME);
  const file = bucket.file(gcsPath);

  const safeName = filename.replace(/"/g, '\\"');
  const [url] = await file.getSignedUrl({
    version: 'v4',
    action: 'read',
    expires: Date.now() + expiresInMinutes * 60 * 1000,
    responseDisposition: `attachment; filename="${safeName}"`,
  });

  return url;
}
```
