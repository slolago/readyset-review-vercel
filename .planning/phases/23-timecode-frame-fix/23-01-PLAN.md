<objective>
Fix the SMPTE timecode frame number not updating when stepping frame-by-frame.

Purpose: The rAF loop gates setCurrentTime behind a 0.25s threshold. A single frame at 30fps is ~0.033s, so frame-step changes never reach the display. Adding explicit setCurrentTime + onTimeUpdate calls after each frame-step bypasses the threshold.

Output: VideoPlayer.tsx with immediate timecode updates on frame step.
</objective>
