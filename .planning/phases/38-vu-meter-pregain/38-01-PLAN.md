# Plan 38-01: Rewire AnalyserNode to tap audio chain before GainNode

## Goal
VU meter measures the source audio signal, not the post-volume output. Adjusting the volume slider or muting must not affect the meter reading.

## Context
- **Files**: `src/components/viewer/VUMeter.tsx`, `src/components/viewer/VideoPlayer.tsx`
- **Root cause**: `createMediaElementSource` output is affected by `video.volume` and `video.muted`. Analysers connected to `source` read post-volume levels.
- **Fix**: Move volume/mute control from `video.volume`/`video.muted` to a Web Audio `GainNode`; analysers tap `source` before the GainNode → always see full signal

## Implementation

### VUMeter.tsx

1. Add `gainNodeRef`, `volumeRef`, `mutedRef` refs
2. Add `setVolume(v)` and `setMuted(m)` methods that update `gainNodeRef.current.gain.value`
3. Expose `setVolume` and `setMuted` on `VUMeterHandle`
4. Audio chain:
   - `source → gainNode → ctx.destination` (playback at user's desired volume)
   - `source → splitter → aL, aR` (measurement, always pre-gain)
5. Force `video.volume = 1; video.muted = false;` after creating source

### VideoPlayer.tsx

1. In `handleVolumeChange`: add `vuMeterRef.current?.setVolume(v); vuMeterRef.current?.setMuted(v === 0);`
2. In `toggleMute`: add `vuMeterRef.current?.setMuted(v.muted);`
3. Keyboard 'M': add `vuMeterRef.current?.setMuted(v.muted);`

## Verification

1. `npm run build` — zero TypeScript errors
2. Open a video, confirm audio plays
3. Move volume slider to 50% — VU meter levels unchanged
4. Mute player — VU meter still shows signal (measures source, not speaker output)

## Files Changed
- `src/components/viewer/VUMeter.tsx` — GainNode for playback, analysers tap pre-gain source
- `src/components/viewer/VideoPlayer.tsx` — route volume/mute through vuMeterRef
